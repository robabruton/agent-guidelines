# Development Attribution Rules

- Do not include `Co-Authored-By` lines referencing AI tools, code assistants, or automation tools in commit messages
- Do not reference AI tools, LLMs, code assistants, or automation tools as having developed, written, generated, reviewed, or planned any part of the project
- This applies to code comments, documentation, commit messages, README content, changelog entries, and all other committed files
- References to AI or automation are allowed when they describe project functionality, integrations, APIs, or user-facing behavior
- Functional references must not imply that a tool was used as a development author or contributor
- When in doubt, ask
